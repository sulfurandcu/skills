# 嵌入式 C 代码审查规则

你必须依据以下规则审查代码，不可遗漏任何一类。审查范围包括新增行及其上下文（diff 中 `+` 行所在函数的完整逻辑）。

---

## 一、审查维度

### 1. 内存安全（Memory Safety）

| 检查项 | 适用 | 严重度 |
|--------|------|--------|
| 动态分配（malloc/calloc/realloc）后未检查 NULL 返回 | Linux App | 严重 |
| 释放后未置 NULL，存在悬垂指针（use-after-free / double-free 风险） | 全部 | 严重 |
| 栈上大数组（> 512 字节），可能导致栈溢出 | MCU | 严重 |
| 递归调用且无深度限制 | MCU | 严重 |
| 全局缓冲区无并发保护（中断与主循环共享） | MCU | 严重 |
| 内存泄漏：所有 exit 路径是否释放已分配资源 | Linux App | 警告 |
| 变长数组（VLA）在 MCU 堆栈受限场景下使用 | MCU | 警告 |

### 2. 指针与数组安全

| 检查项 | 适用 | 严重度 |
|--------|------|--------|
| 指针解引用前未检查 NULL | 全部 | 严重 |
| 数组访问无边界检查，存在越界风险 | 全部 | 严重 |
| sprintf / strcpy / strcat / gets 等不安全函数 | 全部 | 严重 |
| 字符串操作未确保 null-terminated | 全部 | 警告 |
| 函数返回局部变量地址 | 全部 | 严重 |

### 3. 中断与并发（MCU 重点）

| 检查项 | 适用 | 严重度 |
|--------|------|--------|
| ISR 内执行耗时操作（打印、复杂计算、调用非重入函数） | MCU | 严重 |
| 主循环与 ISR 共享变量未使用 volatile | MCU | 严重 |
| 多字节变量在 ISR 与主循环间共享，读取未关中断保护原子性 | MCU | 严重 |
| ISR 内调用非重入标准库函数（如 printf） | MCU | 警告 |

### 4. 硬件交互（MCU 重点）

| 检查项 | 适用 | 严重度 |
|--------|------|--------|
| 外设寄存器操作未使用 volatile 指针 | MCU | 严重 |
| 写寄存器后缺少必要的等待/同步操作（数据手册要求） | MCU | 警告 |
| 未使能外设时钟即访问外设寄存器 | MCU | 警告 |
| 看门狗未在合适位置喂狗（可能导致超时复位） | MCU | 建议 |

### 5. 资源管理

| 检查项 | 适用 | 严重度 |
|--------|------|--------|
| 打开的文件句柄/socket 在所有退出路径上是否关闭 | Linux App | 警告 |
| 线程/进程创建后未处理失败路径 | Linux App | 警告 |
| 互斥锁获取后未能在异常路径释放 | Linux App | 严重 |

### 6. 错误处理与健壮性

| 检查项 | 适用 | 严重度 |
|--------|------|--------|
| 系统调用/库函数返回值未检查 | 全部 | 警告 |
| 外部输入（串口、网络、传感器）未校验长度/范围 | 全部 | 严重 |
| 除法运算未检查除数为零 | 全部 | 警告 |
| 整数溢出（有符号溢出 UB，无符号回绕） | 全部 | 警告 |

### 7. 嵌入式 C 代码规范

| 检查项 | 适用 | 严重度 |
|--------|------|--------|
| 魔数未定义为有意义的宏或常量 | 全部 | 建议 |
| ISR 函数未使用 `__interrupt` 或平台相关修饰符 | MCU | 警告 |
| 未初始化局部变量直接使用 | 全部 | 警告 |
| 死代码或不可达分支 | 全部 | 建议 |
| 注释与代码实际逻辑不一致 | 全部 | 警告 |

---

## 二、严重度判定标准

| 级别 | 含义 | 提交策略 |
|------|------|----------|
| **严重** | 确定会导致崩溃、死机、数据损坏或安全漏洞 | 阻断提交 |
| **警告** | 特定条件下可能出错，或违反编码规范 | 阻断提交 |
| **建议** | 不影响功能但可提升质量 | 不阻断提交 |

存在任意「严重」或「警告」级别问题时，提交被阻断。仅存在「建议」级别问题时允许提交。

---

## 三、审查流程

1. 通过 review-svn 的 svn_diff 获取变更内容（仅关注 `+` 行）。
2. 对每个有 `+` 行的函数，读取完整函数代码理解上下文。
3. 按上述维度逐项检查。
4. 按模板生成审查报告。

---
